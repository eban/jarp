■ [Ruby] lilyのRuby 1.9対応

改行問題はunzip -aaとするのが簡単。VERSIONをRUBY_VERSIONにというのはまあいいとして、trackback_countがundefined methodになるのが全然わからなかった。 試しにlogger.rbを使ってデバッグ開始。instance_evalが怪しいので 

    File::open(f,"r"){|src|
      begin
        self.instance_eval(src.read, f)
      rescue Exception => e
        require 'logger'
        Logger.new("lily.log").error(e.inspect)
      end
    }

のようにinstance_evalにファイル名を追加して、 どういう例外になっているのか確認。 

# Logfile created on Tue Apr 06 02:35:23 JST 2004 by logger.rb/1.10
E, [2004-04-06T02:35:23.329977 #6242] ERROR -- :
  #<SyntaxError: compile error
  plugin/trackback.rb:20: too short multibyte code string:
    /[\300-\377][\000-\177\300-\377]/
  plugin/trackback.rb:79: too short multibyte code string:
    /[\300-\377][\000-\177\300-\377]/>

なるほど。SyntaxErrorか。だからtrackback_countが未定義のままなんだ。 鬼車がこの表記を嫌っているようなので、//nのように指定してあげればよさそう。

というわけで、 Ruby 1.9対応。

2004-04-06 (Tue)

■ [Ruby] lilyをw3mのlocal cgiで動かす

plugin/*.rbの@urlを@cgi.script_nameに置き換えるとw3mのlocal cgiで動くようになる(最近のw3mはPATH_INFOにも対応してるので)。

いちいちcig-binに置くのが面倒なら 

% w3m -o cgi_bin=$PWD file:/cgi-bin/lily.cgi

とすれば、簡単に試せる($PWDは適当に)。
