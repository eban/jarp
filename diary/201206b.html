2012-06-17 (Sun)

■ Bashのインクリメント

最近では便利なので$(())とか$[]で算術演算をがんがんに使ってんだけど、
インクリメントがなんかおかしい。 

% bash -c 'set -xe; i=0; ((i+=1)); echo $i'
+ i=0
+ (( i+=1 ))
+ echo 1
1
% bash -c 'set -xe; i=0; ((i++)); echo $i'  
+ i=0
+ (( i++ ))
zsh: exit 1     bash -c 'set -xe; i=0; ((i++)); echo $i'

なぜかi++は終了ステータス1を返すようで。
どういうこと？i+=1だって1を返していないというのに。
このせいでset -eが使い辛くなってる。
しょうがないので:でごまかしてる 

% bash -c 'set -xe; i=0; : $((i++)); echo $i'
+ i=0
+ : 0
+ echo 1
1

:は常に終了ステータス0を返すので問題ない。

変なのはポストのときだけみたい。++iや--iだと問題ない。
i++とi--ときはなぜか終了ステータスが1になる。
あれ試してみたらzshでも同じだった。
なんか決まりがあるのかな？

よくよくマニュアルを読んでみると 

式の値が 0 でない場合、返却ステータスは 0 になります。
そうでない場合、 返されるステータスは 1 になります。

そういうことか。逆なんだな。納得した。
ってことは:を使うのは間違ってないわけだ。

2012-06-19 (Tue)

■ torne 4.0と台風

録画中にもかかわらずtorneを更新したら最後に、 

現在、放送中の番組を録画しています。
torneのアップデートを完了するには、一度録画を停止する必要があります。
XMBに戻り、録画を停止するか、録画が終了するのを待ってから再度起動してください。

終了するには、PSボタンを押して「終了する」を選択してください。

とメッセージが出てきた。つまり選択肢は3つあるように思える。
待つのとPSボタンを押すのはいいとしても、
どうやって「録画を停止する」のかがわからない。
PSボタンを押して終了するとXMBに戻るが、
torneを選んじゃうとまたメッセージの画面に戻ってきてしまうし。
電源を切るとかいろいろ試したが、結局のところ待つしかないようで。

しかしこんな時間に録画予約してないんだけど、何を撮っているのか？
台風情報だった。NEWS7がこんなに延長してるんかい。

まさか台風の影響でtorneのアップデートを邪魔されようとは。

nasneの宣伝がうぜ〜。
