2009-10-15 (Thu)

■ [Golf] Count diamonds level 1

3つの部分からなるわけでsplitした結果のサイズを3で割ればいいわけだよなあ。
*が5こあるから*を数えて5で割るのと**の個数というのは気づいたんだけど。

最初はuniqを使った。 

grep -o '*\*\|-'|uniq -c|awk '$0*=/\*/'

grep -oはgolfで結構役立つ。

これ はうまいな。

2009-10-16 (Fri)

■ [Tips] Function to output an ASCII character given its decimal equivalent

これは元々 

chr() { echo -en "\0$(( $1 % 8 + 10 * ( $1 / 8 ) + 20 ))"; }

となっていた。63以下がだめだめだしどうなんだと思ったら、いつのまにか 

chr() { printf \\$(($1/64*100+$1%64/8*10+$1%8)); }

に置き換えられていた。まあ、でも発想は面白い。

それにしても

My octal conversion formula was completely wrong.

と言われてもなあ。別物として登録して欲しかった。

100倍10倍とするよりは 

chr() { printf \\$[$1/64]$[$1%64/8]$[$1%8]; }

で並べちゃったほうがわかりやすいかもしれない。
